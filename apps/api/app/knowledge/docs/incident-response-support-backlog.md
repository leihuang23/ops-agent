# Incident Response - Support Backlog Spike

A support backlog spike is a symptom unless it can be tied to a specific product or billing failure. Use ticket categories and affected accounts to avoid over-attributing revenue impact.

## Triage

- Separate billing, product, integration, account, and performance tickets.
- Count high-priority open tickets by scenario and account segment.
- Link duplicate tickets before reporting volume.
- Cite ticket IDs for every support claim.
