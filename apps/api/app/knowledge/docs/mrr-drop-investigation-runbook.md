# MRR Drop Investigation Runbook

Start every MRR drop investigation with deterministic analytics before reading narrative notes. The report should separate root cause, contributing factors, symptoms, and unknowns.

## Evidence Order

1. Current and previous paid invoice MRR windows.
2. Failed, void, and canceled subscription evidence by account.
3. Support tickets from affected accounts in the last 30 days.
4. Product events that could explain usage or workflow disruption.
5. Internal docs or incident records that match the observed pattern.

## Reporting Standard

Every major claim needs a citation to a SQL result, ticket, product event group, incident record, or knowledge document chunk. Do not cite documents that were not retrieved.
